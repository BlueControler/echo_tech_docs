# 自定义进度流与场景验收

## 范围

本文记录复杂任务进度的前后端契约与场景验收要求。它刻意复用现有的 LangGraph Agent Server 流式端点与前端 SSE 解析器。

不引入新的传输方式。

## 流式接口

前端调用现有的 LangGraph 运行流式端点：

```http
POST /threads/{thread_id}/runs/stream
```

请求中 `stream_mode` 必须包含 `custom`：

```json
{
  "stream_mode": ["messages-tuple", "updates", "tasks", "custom"],
  "on_disconnect": "cancel"
}
```

`on_disconnect` 必须为 `cancel`。否则 Agent Server 默认使用 `continue`，
前端流连接断开后旧 run 仍会继续执行，并可能在前端重新连接后继续操作手机。

## 自定义进度负载

后端会发出两类 LangGraph `custom` 分片：

- `task_complexity`：任务级复杂度分类，运行开始附近只发送一次。
- `task_progress`：步骤/工具级进度更新，工具执行期间持续发送。

## 任务复杂度负载

后端在每次运行中仅发送一次该分片，位置接近任务处理开始：

```json
{
  "type": "task_complexity",
  "complexity": "complex",
  "trackSteps": true,
  "reason": "multi_step_plan_required"
}
```

必填字段：

- `type`：必须为 `task_complexity`。
- `complexity`：`simple` 或 `complex`。
- `trackSteps`：纯文本回答或预计只需 0/1 次工具调用的任务为 `false`，需要规划为多个相互依赖步骤的任务为 `true`。
- `reason`：当前取值为 `text_only_answer`、`zero_or_one_tool_call` 或 `multi_step_plan_required`。

分类规则：

- `simple`：纯文本回答，或预计可在 0/1 次工具调用内完成的任务。例如一次天气查询、一次系统读取或一次手机观察。
- `complex`：必须规划为多个相互依赖步骤的任务，或预计需要使用多个工具/动作/工具域。

示例：

- `解释一下 LangGraph 是什么` -> `simple`, `trackSteps=false`。
- `查询深圳今天的天气` -> `simple`, `trackSteps=false`。
- `读取当前手机已安装应用列表` -> `simple`, `trackSteps=false`。
- `观察当前手机页面` -> `simple`, `trackSteps=false`。
- `查询深圳天气，并读取当前手机已安装应用列表` -> `complex`, `trackSteps=true`。
- `观察当前手机页面，并点击屏幕上的搜索框` -> `complex`, `trackSteps=true`。
- `打开浏览器，搜索蓝心小V` -> `complex`, `trackSteps=true`。

## 任务进度负载

后端发出的进度分片形如：

```json
{
  "type": "task_progress",
  "label": "tap",
  "status": "running",
  "phase": "phone_tool",
  "message": "Running phone tool: tap",
  "toolName": "tap",
  "currentStep": 2,
  "totalSteps": 5,
  "completedSteps": [
    {
      "index": 1,
      "name": "observe",
      "status": "completed"
    }
  ],
  "error": "optional error summary"
}
```

必填字段：

- `type`：必须为 `task_progress`。
- `label`：当前进度项的稳定展示标签。
- `status`：`started`、`running`、`completed`、`failed` 之一。
- `phase`：来源大类，例如 `phone_tool`、`system_tool`、`external_tool` 或 `agent`。

可选字段：

- `message`：可读的详细说明。
- `toolName`：稳定的工具键，前端用它在进度分片之间更新同一行。
- `currentStep` / `totalSteps`：已知时用于复杂任务的步骤计数。
- `completedSteps`：后端有更高层计划时的已完成步骤摘要。
- `error`：简短失败原因。
